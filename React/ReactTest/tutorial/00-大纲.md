# 《React + TypeScript 卓越测试实战：架构、数据与代码》— 详细大纲

## 序章：测试的思维模型（00-序章.md）

### 学习的起点：我们为什么写测试
- 测试不是「找 bug 的工具」，而是「设计反馈系统」
- 好的测试让重构变得安全，差的测试让任何改动都成为噩梦
- 测试奖杯（Testing Trophy）vs 测试金字塔：前端该优先投资集成测试
- 静态类型（TypeScript）、单元测试、集成测试、E2E 的分工与边界

### 本书的技术栈与选择理由
- React 18+ / TypeScript 严格模式 / Vite
- Vitest 为什么替代 Jest：原生 ESM、零配置 TypeScript、HMR 级 watch、与 Vite 无缝对齐
- React Testing Library 的设计哲学：从用户视角测试，不测试实现细节
- MSW (Mock Service Worker) 的设计哲学：在网络层拦截，让测试运行在「真实」环境中

### 如何使用本书
- 渐进式结构：从单文件单元测试到完整应用集成测试
- 每章的练习与思考题
- 配套代码的使用方式
- 约定：所有代码默认在 Vitest + jsdom + TypeScript strict 模式下运行

### 关键数据：Vitest vs Jest
- 冷启动快 27 倍（0.3s vs 8s）
- 完整套件快 10 倍
- 配置行数减少 65%（18 行 vs 50+ 行）

### 3.3 渲染与断言

### 3.4 常见反模式
- 直接操作 container.querySelector
- 测试 state/props 而非渲染输出
- 过度使用 data-testid
- 在测试中调用组件方法
- 使用 fireEvent 而非 userEvent

## 第五章：MSW — 网络层的模拟哲学（05-MSW网络层模拟.md）

### 5.1 为什么要在网络层 Mock
- 传统 mock 方案的问题：vi.mock('axios') / vi.mock('fetch') 的脆弱性
- 网络层 mock 的优势：替换 HTTP 客户端时测试无需改动
- MSW 的工作原理：Service Worker（浏览器）/ 模块拦截（Node）

### 5.2 MSW v2 核心 API
- http.get / http.post / http.put / http.patch / http.delete
- handler 回调：({ request, params, cookies }) => {}
- HttpResponse 各静态方法：json, text, xml, formData, error
- 状态码与 Headers 设置
- delay 模拟网络延迟
- passthrough 放行请求
- { once: true } 一次性 handler

### 5.3 handler 组织方式
- 按功能域分组（auth.ts, users.ts, products.ts）
- barrel export（index.ts 合并）
- 初始 handler（happy path）vs 运行时 handler（edge cases）
- 进阶：工厂函数创建可配置 handler（createAuthHandlers({ shouldError, delay })）

### 5.4 server 与 worker
- setupServer vs setupWorker 的区别与共同点
- 同一套 handler 同时用于测试和开发
- worker.start() 的时机与竞态条件
- 非 Node 环境下的 worker 注册

### 5.5 运行时覆盖：server.use()
- server.use 的 prepend 行为
- 在单个测试中覆盖特定端点
- 覆盖的优先级与合并行为
- 典型场景：错误 vs 空数据 vs 正常数据
- server.resetHandlers 与 server.restoreHandlers 的区别

### 5.6 GraphQL Mock
- graphql.query / graphql.mutation
- variables 匹配与 response 分支
- 与 Apollo Client / urql 的集成测试
- GraphQL 订阅的当前支持状态（Discussion #2352）

### 5.7 WebSocket 与 SSE Mock
- ws.link() 创建 WebSocket 拦截
- client.send / client.close / broadcast
- 第三方库绑定（@mswjs/socket.io-binding）
- sse() API（v2.12.0+）

### 5.8 对照 MSW v1 的迁移速查表
- rest.get → http.get
- res(ctx.json(x)) → HttpResponse.json(x)
- ctx.delay → delay()
- 完整 import 路径变化

### 练习与思考
