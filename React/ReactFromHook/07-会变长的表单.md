# 第七章：会变长的表单

> 真实项目中到处是"点击添加更多"——多个收货地址、N 段工作经历、标签列表。字段数量在用户操作中动态变化。本章深入 `useFieldArray`——react-hook-form 管理不定数量字段的解决方案。

---

> [上一章](06-接入第三方组件库.md) · [下一章](08-高级模式与技巧.md)

## 7.1 从一个日常场景出发

想一想"添加收货地址"的界面：

```
┌─────────────────────────────┐
│ 地址 1: [________________]  │  [删除]
│ 地址 2: [________________]  │  [删除]
│ 地址 3: [________________]  │  [删除]
│                              │
│ [+ 添加新地址]                │
└─────────────────────────────┘
```

页面加载时只有 1 行。用户可以加到 5 行，也可以删到 0 行。字段数量是**动态的**——不是代码写死的固定数量。

> ==动态字段的本质是一个数组==。你用 state 管理数组，用 map 渲染，用 splice/push 操作。`useFieldArray` 就是把这些操作封装好，确保每次增删改都与 react-hook-form 的内部数据保持同步。你不需要手动维护 `fields` state 和表单数据的一致性——库替你做了。

---

## 7.2 useFieldArray 的 API

```tsx
import { useForm, useFieldArray } from 'react-hook-form';

const { fields, append, remove } = useFieldArray({
  control,  // 从 useForm 拿的——useFieldArray 需要接入表单的控制系统
  name,     // 这个数组叫什么名字——对应 defaultValues 里的 key
});
```

完整操作集：

| 操作 | 函数 | 说明 |
|------|------|------|
| 追加 | `append({ name: '' })` | 在末尾加一行 |
| 前置 | `prepend({ name: '' })` | 在开头加一行 |
| 插入 | `insert(2, { name: '' })` | 在索引 2 位置插入 |
| 删除 | `remove(1)` | 删除索引 1 |
| 交换 | `swap(0, 2)` | 交换索引 0 和 2 |
| 移动 | `move(0, 3)` | 把索引 0 移到索引 3 |
| 替换 | `replace([...])` | 整个数组替换 |

## 7.3 完整示例：标签编辑器

```tsx
function TagsEditor() {
  const { register, control, handleSubmit } = useForm({
    defaultValues: {
      tags: [{ name: '' }], // 初始一行，页面不会空荡荡
    },
  });

  const { fields, append, remove } = useFieldArray({
    control,
    name: 'tags',
  });

  const onSubmit = (data) => console.log(data.tags);
  // 输出：[{ name: 'React' }, { name: 'TypeScript' }]

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {fields.map((field, index) => (
        <div key={field.id}>
          <input
            {...register(`tags.${index}.name`, {
              required: '标签不能为空',
            })}
            placeholder={`标签 ${index + 1}`}
          />
          <button type="button" onClick={() => remove(index)}>
            删除
          </button>
        </div>
      ))}

      <button type="button" onClick={() => append({ name: '' })}>
        + 添加标签
      </button>

      <button type="submit">提交</button>
    </form>
  );
}
```

### 逐行解读

- **`defaultValues: { tags: [{ name: '' }] }`**——初始一行空标签。如果不给初始行，页面打开时是空的，用户需要先点"添加"
- **`` register(`tags.${index}.name`) ``**——用模板字符串拼接路径。第 0 行的 name 字段路径是 `tags.0.name`
- **`key={field.id}`**——==必须用 `field.id`==，不能拿 `index` 当 key（下一节解释为什么）
- **`append({ name: '' })`**——追加一行空标签，结构匹配 defaultValues 中的单行结构

## 7.4 为什么 key 不能用 index

这是 React 动态列表的经典陷阱，这里出错的后果比其他场景更严重。

假设有 3 行：

```
[0] React        [删除]
[1] TypeScript   [删除]
[2] Vue          [删除]
```

删除 "TypeScript"（index=1），数组变成：

```
[0] React        [删除]
[1] Vue          [删除]    ← Vue 的 index 从 2 变成了 1
```

用 `key={index}` 时，React 看到 key=0 和 key=1 还在，以为这两行没变——不会重新挂载。但实际上 key=1 对应的内容从 TypeScript 变成了 Vue。结果是**input 里的值和显示位置错位**——TypeScript 的 DOM 节点被复用来显示 Vue，用户看到的内容和实际数据对不上。

`field.id` 是稳定的唯一标识——删除操作不会改变其他行的 id：

```tsx
// ✅ 正确
{fields.map((field, index) => (
  <div key={field.id}>...</div>
))}

// ❌ 错误——这是 bug 的来源
{fields.map((field, index) => (
  <div key={index}>...</div>
))}
```

## 7.5 带 Zod 校验的动态字段

```tsx
const schema = z.object({
  tags: z.array(
    z.object({
      name: z.string().min(1, '标签不能为空').max(20, '最多 20 个字符'),
    })
  ).min(1, '至少需要一个标签').max(10, '最多 10 个标签'),
});

// Controller 版——用于第三方 UI 组件
<Controller
  name={`items.${index}.name`}
  control={control}
  render={({ field, fieldState }) => (
    <>
      <input {...field} />
      {fieldState.error && <span role="alert">{fieldState.error.message}</span>}
    </>
  )}
/>
```

数组本身的校验（`.min(1)`、`.max(10)`）和数组元素内部的字段校验可以同时存在——Zod 和 react-hook-form 各自负责自己那层。

## 7.6 嵌套数组

数组里面还能有数组——比如"项目组"里有"多个成员"：

```tsx
// 数据结构
{
  groups: [
    {
      name: '前端组',
      members: [
        { name: '张三' },
        { name: '李四' },
      ],
    },
  ],
}

// 内层 useFieldArray
const { fields: groupFields, append: appendGroup } = useFieldArray({
  control,
  name: 'groups',
});

// 每个 group 里再独立管理它的 members
const { fields: memberFields, append: appendMember } = useFieldArray({
  control,
  name: `groups.${groupIdx}.members`, // 注意路径：点到具体 group 下的 members
});

// 注册路径用点号串联到底
register(`groups.${groupIdx}.members.${memberIdx}.name`)
```

路径嵌套的规则：就是普通的 JavaScript 对象路径，用 `.` 串联。没有魔法。

## 本章要点

- `useFieldArray` 管理==行数不固定==的字段列表，本质是数组操作的封装
- 增删改操作：`append`、`prepend`、`insert`、`remove`、`swap`、`move`、`replace`
- map 渲染时 key **必须**用 `field.id`——==用 index 会导致 DOM 和数据的错位==
- 注册路径：`` register(`数组名.${index}.字段名`) ``——用模板字符串拼接
- 嵌套数组路径就是普通的 `.` 链式拼接，没有特殊规则

## 思考题

1. 如果你删除了数组中间的一项，`fields` 数组的长度和索引会立即更新。但如果你同时有 `watch` 订阅了这个数组，watch 的回调是在删除前还是删除后触发？为什么这很重要？
2. `useFieldArray` 的 `append` 可以在新行里设置默认值。如果新行的默认值依赖于已有行的值（比如默认继承上一行的某些字段），你会怎么实现？
3. 一个表单同时有 3 个独立的 `useFieldArray`（比如教育经历、工作经历、技能标签）。它们各自独立还是共享底层的表单状态？如果删光一个数组的所有行，提交时 data 里这个数组是 `[]` 还是 `undefined`？
