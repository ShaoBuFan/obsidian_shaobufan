# react-hook-form：从第一性原理到工程实践

> 这不是一本 API 手册。这是一次思维的重新布线——用第一性原理拆解表单问题，用费曼技巧让每个概念透明。

## 这门课解决什么问题

你在写 React 表单时是否遇到过这些困扰：
- 30 个字段的表单，输入时卡顿明显
- 校验逻辑散落在几十个 JSX 里，改一个规则要找半天
- 第三方组件库接入表单时写法诡异，网上教程互相矛盾
- 动态增删字段时，key、索引、状态全乱套

这背后的根本原因不是"你还不够熟练"。而是你一直在用**受控组件的思维**写表单——每次按键都触发整个表单的重渲染。这门课从根上解决这个认知问题。

## 教学哲学

本课程将三种教学方法**融合为一个连贯的叙事**：

**第一性原理**把问题拆到不能再拆——"没有这个工具会怎样？它到底解决了什么根本问题？"每一章都从这个原点出发。

**费曼技巧**要求把复杂概念翻译成简单语言——"如果我讲不清楚，说明我还没真懂。"本课程所有解释都通过这个检验。

**渐进式脚手架**让学习曲线尽可能平缓——你的每个新知识都稳稳站在前面的基础上。

你不会在这些页面里看到"第一性原理分析""费曼讲解"这样的章节标题。它们已经融进了每一段话里。

## 谁可以学

- 用过 React 写过页面，知道 `useState`、`useEffect` 是什么
- 不需要事先了解 TypeScript（但强烈建议配合学习）
- **不需要认识 react-hook-form——从零开始**
