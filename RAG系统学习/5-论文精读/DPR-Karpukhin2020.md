# DPR · Dense Passage Retrieval for Open-Domain QA

Karpukhin et al. (Facebook AI), EMNLP 2020. 稠密检索的奠基之作。

## 核心问题

开放域问答的传统方案是 BM25 → 检索相关文档 → 阅读理解模型抽取答案。BM25 靠精确词匹配，无法处理语义相似但用词不同的情况（"心脏不舒服" vs "心肌梗死"）。能否用稠密向量检索替代 BM25，不依赖任何特定任务的监督信号，达到甚至超越 BM25 + 重排的效果？

## 方法

==Dual-encoder 架构==：两个独立的 BERT 编码器。

- **Query Encoder**：编码查询。短文本，BERT-base，取 `[CLS]` token 输出。
- **Passage Encoder**：编码文档片段。可以很长，BERT-base，取 `[CLS]` token 输出。
- **相似度**：`sim(q, p) = q · p`（两个向量的点积）。

> 注意：DPR 用点积而非余弦相似度。实际上因为向量做了 L2 归一化，点积 = 余弦。这和你在 [相似度度量体系](../1-理论基础/2-相似度度量体系.md) 中学到的是一回事。

**训练关键是负例选择：**

1. **Random negatives**：从语料库中随机采样——太容易了，模型不需要学精细语义区分
2. **BM25 negatives**：用 BM25 检索到的但不包含答案的段落——大部分是不相关的，少数碰巧部分相关
3. **==In-batch negatives==**：同一个 batch 中其他问题的正例段落作为当前问题的负例。免费，因为 batch 内 passage 的向量本来就计算了，直接复用。训练效率提高 N 倍（N = batch size）
4. **Gold negatives（hard negatives）**：从同一个 batch 中选 BM25 检索出的且不包含答案但和正例在同一 batch 的段落。这是最难的负例，对模型学习最有用

## 关键实验

**主要结果（Natural Questions, Top-20 准确率）：**

| 方法 | Top-20 |
|------|--------|
| BM25 | 59.1 |
| DPR (仅 in-batch negatives) | 78.4 |
| DPR (+ gold hard negatives) | **79.4** |
| BM25 + DPR 混合 | 76.9（不如纯 DPR） |

> 反直觉：BM25 + DPR 混合在这个数据集上不如纯 DPR。不是混合检索不好——是当稠密检索本身足够强时，BM25 的噪声反而拖累融合效果。这说明混合检索的收益取决于两个检索器的**质量差**，不是任何情况下混合都更好。

**泛化能力**：在 Natural Questions 上训练的 DPR，直接在 WebQuestions、CuratedTREC 等数据集上 zero-shot 测试，结果显著优于 BM25。==稠密检索的泛化能力远超很多人的预期==。

## 优势与局限

**优势：**
- 建立了 dual-encoder → 点积 → top-K 的标准范式，至今仍是稠密检索的默认架构
- In-batch negatives 是一种"免费午餐"式的训练优化，后续工作广泛采用
- 证明了 BM25 不是不可超越的 baseline——在语义匹配场景下，稠密检索可以大幅领先

**局限：**
- Dual-encoder 的本质弱点：查询和段落的交互只在最后的点积环节发生，没有 token 级别的交互。这意味着"精细匹配"（如精确数字、代码片段）不如 BM25
- `[CLS]` token 作为段落表示在长文本上的效果有争议——后续工作发现 mean pooling 往往更好（Sentence-BERT 的选择）
- 领域迁移：在 NQ 上训练的 DPR 到生物医学文献上效果显著下降

## 对本项目的启发

1. **In-batch negatives 的智慧**：不需要专门构造负例集，batch 内复用。向量记忆库如果用领域数据 fine-tune 嵌入模型，这个技巧可以显著降低数据准备成本。

2. **BM25 + 稠密不是"越多越好"**：DPR 的实验表明，当你有一个足够好的稠密检索器时，强行加入 BM25 可能降低效果。你的向量记忆库迁移到 BGE-M3 后应该**对比测试**纯稠密 vs 混合检索，不要假设混合一定更好。

3. **验证了你在 [稠密检索](../2-索引与检索/5-稠密检索.md) 中的结论**：领域差距（domain gap）是稠密检索的"无声杀手"。DPR 从 NQ 迁移到专业领域时性能下降——你的中文哲学笔记同理。
