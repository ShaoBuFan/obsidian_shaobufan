# 必学工具

> 程序员的瑞士军刀。不需要全部精通，但至少要知道它们能干什么、什么时候用。

## 工具清单

| 工具 | 用途 | 优先级 |
|------|------|--------|
| **翻墙** | 访问 Google、GitHub、StackOverflow | 🔴 必须 |
| **Git** | 版本控制，所有代码的时光机 | 🔴 必须 |
| **GitHub** | 代码托管、协作、找项目 | 🔴 必须 |
| **Vim** | 终端编辑器，服务器上改配置必备 | 🟡 推荐 |
| **Docker** | 环境隔离，告别"在我机器上能跑" | 🟡 推荐 |
| **LaTeX** | 论文排版（非本科生可暂缓） | 🟢 按需 |
| **GNU Make** | 构建自动化 | 🟡 推荐 |
| **CMake** | C/C++ 构建系统 | 🟡 推荐 |
| **Scoop** | Windows 包管理 | 🟢 按需 |
| **信息检索** | Google 搜索技巧、文献检索 | 🔴 必须 |
| **Emacs** | 另一款经典编辑器 | 🟢 按需 |

## 学习顺序

1. 翻墙 + 信息检索 + Git + GitHub — 第一个周末搞定
2. Vim 基础 — 学会打开、编辑、保存、退出（`:wq` 不难）
3. Docker — 学操作系统或做项目时自然用到
4. LaTeX — 写论文前学

## 不需要做的事

- 把 Vim/Emacs 配成 IDE（用 VS Code / JetBrains 写代码就行）
- 一开始就学 Make/CMake（写小项目用不到）
