# ReAct Agent 系统学习

## 为什么研究这个主题

[Function Calling 系统学习](../FunctionCalling系统学习/0-模块索引.md) 研究了模型"怎么调用工具"——API 参数、流式处理、Schema 约束。但那个系列回避了一个更根本的问题：==模型**如何决定**调用哪个工具、在什么时候调用、调用完之后下一步做什么？==

这就是 Agent 推理架构要解决的问题。ReAct（Reasoning + Acting，2022）是这个领域的基础论文——它用一个简单的 prompt 模式让模型学会了"边想边做"。从它开始，衍生出了 Plan-and-Solve、Reflexion、Tree-of-Thought、CodeAct 等一系列架构。

> ReAct 之于 Agent，就像 TCP 之于网络——它是地基协议，后来的所有架构都是在它之上修修补补。
