# ReAct 论文精读（Yao et al., 2022）

## 元数据

- **标题**：ReAct: Synergizing Reasoning and Acting in Language Models
- **作者**：Shunyu Yao, Jeffrey Zhao, Dian Yu, Nan Du, Izhak Shafran, Karthik Narasimhan, Yuan Cao
- **发表**：arXiv:2210.03629，2022 年 10 月；接收于 **ICLR 2023**
- **代码**：github.com/ysymyth/ReAct
- **项目页**：react-lm.github.io
- **模型**：PaLM-540B，仅 1-6 个 in-context examples（==无微调==）

## 核心问题

纯推理（Chain-of-Thought）会在缺乏外部信息时产生幻觉。纯行动（如 WebGPT 的行为克隆）缺乏高层次推理和动态调整能力。问题是：**能否让推理和行动互相增强，而非各自为战？**

## 方法

ReAct 提示范式要求模型交替生成以下三类内容：

```
Thought:  对当前状态的推理，决定下一步做什么
Action:   工具调用（Search / Lookup / Finish）
Observation: 工具返回的结果（由环境提供，不是模型生成的）
```

一个典型的 ReAct 轨迹（HotpotQA）：

```
Question: What is the elevation range for the area that the eastern 
sector of the Colorado orogeny extends into?

Thought 1: I need to search Colorado orogeny, find the eastern sector,
then find the elevation range.
Action 1: Search[Colorado orogeny]
Observation 1: The Colorado orogeny was an episode of mountain building
in Colorado and surrounding areas.

Thought 2: It doesn't mention the eastern sector. I need to look up
"eastern sector" specifically.
Action 2: Lookup[eastern sector]
Observation 2: (Result 1 / 1) The eastern sector extends into the
High Plains and is called the Central Plains orogeny.

Thought 3: The eastern sector extends into the High Plains. I need to
find the elevation range of the High Plains.
Action 3: Search[High Plains elevation range]
Observation 3: High Plains refers to ... rises from 1,800 to 7,000 ft.

Thought 4: Elevation range is 1,800 to 7,000 ft. I can answer now.
Action 4: Finish[1,800 to 7,000 ft]
```

与 CoT 的关键区别：Thought 不是最终答案的推理草稿，而是==持续与外部信息交互的策略决策==。

## 关键实验

### 1. 知识密集型推理（HotpotQA, Fever）

**HotpotQA**（需要跨多个维基页面搜索和推理）：

| 方法 | 准确率 |
|------|--------|
| Standard（无推理） | 28.7% |
| CoT（只推理） | 29.4% |
| Act（只行动） | 28.9% |
| **ReAct** | 33.1% |
| **ReAct + CoT-SC**（Self-Consistency 集成） | **35.1%** |

**Fever**（事实验证）：

| 方法 | 准确率 |
|------|--------|
| Standard | 57.1% |
| **ReAct + CoT-SC** | **64.6%** |

> 关键发现：ReAct 单独使用在 HotpotQA 上略优于 CoT（33.1% vs 29.4%），但 CoT 在 Fever 上单独表现更好（因为 Fever 更多依赖模型内部知识）。**两者结合（ReAct + CoT-SC）在全部基准上取得最佳结果**——内部推理和外部检索互相补充。

### 2. 交互式决策（ALFWorld, WebShop）

**ALFWorld**（文本游戏，需要多步交互）：

| 方法 | 成功率 | 训练数据 |
|------|--------|----------|
| BUTLER（专用 Agent） | 37% | 100K+ 专家演示 |
| **ReAct（2-shot, PaLM-540B）** | **71%** | **仅 2 个示例** |

> 这个结果非常重要：ReAct 仅用 2 个示例就超过了用 10 万条专家数据训练的专用 Agent。==大模型的常识 + ReAct 的推理行动循环 = 零样本/少样本决策能力==。

**WebShop**（网页导航购物）：

| 方法 | 成功率 |
|------|--------|
| IL + RL（模仿+强化学习） | 29.1%（~90K 条训练数据） |
| **ReAct（1-shot）** | **40%** |

### 3. ReAct + CoT 的结合方式

论文实验了三种结合方式：

1. **ReAct → CoT**：ReAct 轨迹结束后，如果信息不够，回退到 CoT 内部推理
2. **CoT → ReAct**：CoT 推理发现信息不足 → 切换 ReAct 模式检索
3. **交替**：每个步骤内部同时做 ReAct 和 CoT，取一致性更好的

> 方式 3 效果最好，但开销最高。方式 2（CoT 触发 ReAct）在 Fever 上接近单独的 CoT 效果——当内部知识足够时，ReAct 的检索反而是噪音。

## 优势与局限

### 优势
- **零样本泛化**：冻结的 PaLM-540B + 1-6 个示例就能在多个领域超越专用系统
- **可解释性**：每一步的 Thought 是自然的推理链，人类可阅读和审计
- **幻觉抑制**：Action 和 Observation 将推理锚定在外部事实上，幻觉概率大幅降低
- **简单**：不需要微调、不需要 RL、不需要修改模型架构——就是精心设计的 prompt

### 局限
- **Token 开销**：每步 Thought 都是额外的输出 token，4× 于 Direct prompting
- **延迟**：串行循环（Thought → Action → Observation → Thought），不可并行化
- **格式脆弱性**：输出的 Thought/Action 是自由文本，需要正则解析，格式错误率高
- **受限于 Prompt 工程**：1-6 个示例的质量直接决定轨迹质量，prompt 质量的方差很大
- **无法处理复杂多步规划**：ReAct 每步只看下一步，没有全局规划能力（这也是 Plan-and-Solve 和 Reflexion 后来要解决的问题）

## 对本项目的启发

ReAct 是本系列 [Function Calling 系统学习](../FunctionCalling系统学习/0-模块索引.md) 的自然前置。Function Calling 解决了"如何表达 Action"的问题——用结构化 JSON 替代自由文本。但 "何时行动、如何根据观察调整策略" 这些决策层的问题，仍然是 ReAct 的核心贡献。

> 2026 年的 Agent 系统本质上是 **ReAct 的决策框架 + Function Calling 的执行机制 + MCP 的工具标准化**的三层叠加。理解 ReAct 原文是理解这个三层架构的基础。
