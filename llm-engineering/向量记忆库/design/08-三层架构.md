# 三层架构

## 纯向量检索的局限

每次对话只检索 top-K（5-10 条）和当前查询最相关的片段，99.9% 的"记忆"用不上。AI 了解的是"这次检索到的你"，不是"你"。

> 更接近"记忆"的路径：向量检索 + 定期摘要 + 用户画像。三层互补。

## 层 1：向量检索（实时）

**做什么**：当次对话中，根据查询语义检索最相关的历史片段。

**实现**：[04-MCP服务设计](04-MCP服务设计.md) 中的 `search_notes` + [07-记忆权重与复合评分](07-记忆权重与复合评分.md) 的 rerank。

**局限**：被动、碎片化、受 top-K 限制。

## 层 2：定期摘要（离线）

**做什么**：每隔一段时间（每周/每月），把新的对话和笔记**压缩为结构化摘要**，而不是保留原始片段。

**和向量检索的关键区别**：

```
向量检索：存原始文本 → 检索时拼原文
定期摘要：存压缩后的摘要 → 检索时拿的是"理解后的版本"
```

**实现思路**：

```python
def periodic_summarize(since_date: str):
    """把最近的新内容分批喂给 LLM 做摘要"""
    new_chunks = collection.get(where={"date": {">=": since_date}})

    # 按主题聚类
    topics = cluster_by_similarity(new_chunks)

    # 每个主题生成一段摘要
    for topic in topics:
        summary = llm.summarize(
            f"用户近期关于{topic}的讨论和笔记：\n" +
            "\n".join(c["text"] for c in topic["chunks"]),
            instruction="提炼核心观点、偏好变化、反复出现的模式。不超过300字。"
        )
        # 摘要作为新的"高密度 chunk"写回向量库
        collection.add(
            documents=[summary],
            metadatas=[{"type": "summary", "topic": topic["name"],
                        "date_range": f"{topic['start']}~{topic['end']}"}],
            ids=[f"summary/{topic['name']}/{today}"]
        )
```

**效果**：一条 300 字的摘要承载了十几次对话的核心信息，检索效率远高于原始片段。

**节奏**：每月跑一次。如果对话量很大（每天几十轮），改为每周。

## 层 3：用户画像（持久）

**做什么**：维护一个**每次对话都完整加载**的用户画像文件。不依赖检索——Claude 一开始就知道你是谁。

**包含什么**：

```markdown
# 用户画像

## 技术偏好
- 偏好简洁代码，厌恶过度设计
- 函数式风格 > 面向对象
- 先理解原理再动手

## 当前关注
- 向量记忆库搭建（2026-05 至今）
- 软件设计的哲学学习（2026-05 至今）

## 性格特征
- 喜欢想清楚再动手
- 对工具有审美要求
- 会追问到底层原理

## 关键经历
- ...（从定期摘要中提炼的重要节点）
```

**和层 2 的关系**：定期摘要发现"反复出现的模式" → 提炼进用户画像。用户画像是摘要的**再压缩**。

**更新方式**：

```python
@server.tool("update_profile")
def update_profile(insight: str, category: str):
    """将一条新洞察写入用户画像"""
    profile_path = "向量记忆库/用户画像.md"
    profile = open(profile_path).read()
    # 在对应 category 下追加 insight
    updated = inject_insight(profile, category, insight)
    open(profile_path, "w").write(updated)
```

**加载方式**：在 `CLAUDE.md` 中引用用户画像文件，Claude Code 每次对话自动读取。

## 三层协作示意

```
对话结束
    │
    ▼
层1 向量库 ← 原始片段实时写入
    │
    │ （每月一次）
    ▼
层2 摘要引擎 ← 聚类 → LLM 压缩 → 摘要写入向量库
    │
    │ （持续提炼）
    ▼
层3 用户画像 ← 模式识别 → 洞察追加 → CLAUDE.md 加载
    │
    ▼
下次对话开始时，Claude 已经知道：
  - 你是谁（层3 完整加载）
  - 深入的话题（层2 高密度摘要可检索）
  - 具体细节（层1 原始片段可检索）
```

## 渐进路径

不用一次性全做。分阶段：

1. **先跑通层 1** —— 向量检索能工作，笔记和对话可查
2. **加层 3 的最简版本** —— 手动写一个用户画像文件，挂到 CLAUDE.md。迭代几周，看哪些信息真的有用
3. **加层 2** —— 当月度摘要跑通后，用摘要提炼的结果反过来更新用户画像
