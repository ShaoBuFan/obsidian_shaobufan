# LLM 系统工程学习路线

## 路线图

从 LLM 的"外部信息接入"到"自主决策"，四个项目形成一条自然的能力跃迁线：

```
第一阶段          第二阶段            第三阶段             第四阶段
(基础)           (实践)             (机制)              (架构)

RAG系统学习  →  向量记忆库    →  FunctionCalling  →  ReAct系统学习
检索增强生成     RAG的工程实现     系统学习              Agent推理架构
                                    工具调用机制
   "喂资料"       "做系统"         "调工具"            "做决策"
```

### 为什么是这个顺序

1. ==RAG 是地基==。在理解 Function Calling 的 Action 之前，需要先理解"LLM 如何获取外部信息"。RAG 解决的检索问题——embedding、ANN 索引、混合检索——是后续一切外部数据交互的基础。

2. ==向量记忆库是把理论变成代码==。RAG 学到的切分策略、检索算法、重排序，在这里落地为可运行的 Python 实现。这个项目的设计文档（`design/00-08`）是 RAG 理论的结构化映射。

3. ==Function Calling 是从"被动检索"到"主动调用"的跨越==。RAG 是"模型被动接收外部信息"——你检索好，塞进上下文，模型读。Function Calling 是"模型主动请求外部信息"——模型决定什么时候查、查什么、查完怎么用。这个自主性的跃迁是 Agent 的基础。

4. ==ReAct 是从"单次调用"到"多步决策"的跨越==。Function Calling 告诉你模型**如何**调用工具，ReAct 告诉你模型**何时**调用工具、**调用之后怎么调整策略**、**何时停止**。ReAct 的 `while` 循环包裹了 Function Calling 的单次调用，形成了完整的 Agent 闭环。

## 第一阶段：RAG 系统学习

**核心问题**：如何让 LLM 获取训练数据之外的信息？

**关键概念**：向量嵌入 → 相似度度量 → ANN 索引 → 文档解析 → 切分策略 → 稠密/稀疏检索 → 混合检索 → 重排序 → 查询改写 → 生成与评估

**入口**：[RAG系统学习/RAG系统学习.md](RAG系统学习/RAG系统学习.md)

**关联工程**：[向量记忆库](../向量记忆库/design/00-总览.md)（RAG 的实践落地）

**预计时间**：4-6 周（理论学习）+ 2-4 周（工程实战）

**关键概念**：工具定义（JSON Schema）→ tool_choice 控制 → 流式处理（SSE）→ 并行调用 → MCP 协议 → PTC 代码级编排 → Prompt Injection 防护

## 第四阶段：ReAct 系统学习

**核心问题**：模型如何**决定**调用哪个工具、何时调用、调用之后下一步做什么？

**关键概念**：ReAct 循环（Thought → Action → Observation）→ Plan-and-Solve → Reflexion → Tree-of-Thought → 2026 演进（Pre-Act、Co-ReAct、半形式化推理、CodeAct）

**入口**：[ReAct系统学习/0-模块索引.md](ReAct系统学习/0-模块索引.md)

**前置知识**：Function Calling（ReAct 的 Action 步骤通过 Function Calling 执行）

**关联工程**：[ReAct 最小实现](ReAct系统学习/3-工程实践/1-ReAct最小实现.md)（三个版本的 Agent 循环代码）

**预计时间**：1-2 周

## 学习建议

- **按顺序，但不必等到"学完"再前进**。RAG 学到第二阶段（索引与检索）后就可以开始 Function Calling——两者的核心概念不冲突。
- **工程实践驱动理论学习**。每学完一个概念笔记，立刻在向量记忆库项目中找到对应的设计文档或代码，做一个小的改进。学完切分策略→优化向量记忆库的 chunker；学完重排序→实现复合评分的 rerank 逻辑。
- **思考题是真的要思考的**。每个概念笔记末尾的 2-3 道思考题不是装饰——它们是"从读懂了到能用出来"的桥梁。如果你不能用自己的话回答，说明还没真正理解。
- **交叉引用是你的地图**。笔记之间的大量交叉引用不是为了凑字数——它们标记了概念之间的依赖关系。顺着引用链走，你能看到单个笔记里看不到的全局图景。
