# CLAUDE.md

This file provides guidance to Claude Code when working with code in this repository.

## 目录用途

RAG（检索增强生成）的系统化学习笔记。从向量嵌入的数学本质到生产级系统的工程权衡，覆盖 RAG 管道的全部环节。

## 笔记风格

- 费曼学习法和第一性原理推导**融合为连贯叙述**，不分区罗列，不出现"第一性原理"、"费曼法"之类的标签
- 第一性原理作为**推导方法**：每篇笔记从不可再简化的根本约束出发——"如果这个技术不存在，什么会崩溃？"——然后沿着"为什么"的链条往上重建，直到触及实践细节。整个叙事就是答案，不需要标注"这里用的是第一性原理"
- 核心思想和实用 tips 使用 `>` 引用块承载
- 不使用 emoji
- 中文为主，每篇概念笔记末尾设思考题
- 技术分析笔记可使用对比表、代码块、ASCII 架构图

## 关键规则

- **必须搜索验证**：关键概念、算法细节、模型参数、基准数据必须通过 WebSearch 搜索交叉验证，不能仅依赖模型训练数据
- 搜索关键词中英双语，优先查找官方文档（ChromaDB、LlamaIndex、LangChain、RAGAS 等）、权威博客（Pinecone、Weaviate、Lilian Weng 等）和 arXiv 论文
- 搜索结果与训练数据有出入时，以搜索结果为准
- 当讨论的算法/工具有明确版本号时，必须注明版本

## 笔记分类与格式

### 概念笔记（`1-理论基础/`、`2-索引与检索/`、`3-生成与评估/`）

- Feynman 叙事风格：从直觉到精确
- `==高亮==` 关键术语
- `>` blockquote 承载核心洞察
- 末尾设 2-3 个思考题
- 关联阅读指向本文件夹内其他笔记或向量记忆库设计文档

### 模块索引（各模块的 `N-模块名.md`）

- 表格列出主题、状态、核心问题
- 关联项目资源（向量记忆库 design/ 文档）
- 学习进度复选框

### 论文精读（`5-论文精读/`）

- 结构：核心问题 → 方法 → 关键实验 → 优势与局限 → 对本项目的启发
- 控制在 1000-1500 字
- YAML frontmatter 加 `paper: author_year` 字段

### 工程实战（`6-工程实战/`）

- 执行清单，不是纯文档
- 每篇对应一个具体的代码实现任务
- 末尾设 `Lab Check: [ ] 已完成  [ ] 已验证`

## Obsidian 格式规范

- **链接**：标准 markdown 链接 `[text](path.md)`，**禁止** wikilinks
- **标签**：顶层 `RAG系统学习`，嵌套用 `/` 如 `概念/嵌入`、`概念/混合检索`
- **frontmatter**：至少含 `tags`（YAML 列表）和 `created`（YYYY-MM-DD）
- **高亮**：`==text==`
- **分隔**：`---` 用作章节分隔（不在文件最顶部单独出现）
- **注释**：`%%text%%`
- **不使用**：脚注、emoji、HTML 标签、callout 语法
- **strictLineBreaks**：段落间留空行
