# 2.7 · 重排序

初检（ANN + 混合检索）在百万级向量上做粗筛，返回 Top-50。这 50 个都相关，但谁排第一？初检的余弦相似度只能判断"整体方向是否接近"——两段文字讨论同一个主题但角度完全相反，余弦相似度可能很高。"深模块好"和"深模块不好"结构相似、用词相似、只差一个"不"字——向量方向几乎一致，但一个是对用户有用的答案，另一个是干扰。

重排序就是在初检的 50 个候选项上做精细排序。因为候选项少（50 而非 1,000,000），可以用更重的算法。

初检用 dual-encoder——查询和文档独立编码，只在最后的点积处交互。重排用 cross-encoder——查询和文档**拼接后一起编码**，查询的每个词都能和文档的每个词做 cross-attention。精度高得多，但每对查询-文档都要跑一次完整前向传播——50 次可以接受，100 万次不行。

> 初检的 dual-encoder 保速度，重排的 cross-encoder 保精度。两者分工：初检不遗漏（高召回），重排不乱排（高精度）。

除了 cross-encoder，还有更轻的重排方案。你的向量记忆库设计了一个不依赖外部 API 的复合评分方案：在余弦相似度的基础上叠加三个自定义维度——深度（chunk 被提及/引用的次数）、最近性（指数衰减，半衰期 30 天）、重要性（显式标记）。`final = semantic·w1 + depth·w2 + recency·w3 + importance·w4`。零额外 API 调用，完全可解释，但需要维护元数据和实验调权重。

重排之后的"最后一公里"：按什么顺序把 chunks 放进 prompt？直觉是相关性从高到低。但 "Lost in the Middle"（Liu et al., 2023）表明 LLM 对上下文的开头和结尾注意力最强，对中间显著衰减。所以最优排列可能是一个 U 型——最相关的放开头和结尾，次要的放中间。
