# RAG 系统学习

检索增强生成（Retrieval-Augmented Generation）的系统化学习路线。目标是深入理解 RAG 的每一层——从向量嵌入的数学本质到生产级系统的工程权衡——并最终完成 [向量记忆库](../向量记忆库/design/00-总览.md) 项目中已设计但未实现的工程缺口。

> 本笔记是 [LLM 系统工程学习路线](../00-学习路线.md) 的第一阶段。学习完 RAG 后，建议继续 [Function Calling 系统学习](../FunctionCalling系统学习/0-模块索引.md)（工具调用机制）→ [ReAct 系统学习](../ReAct系统学习/0-模块索引.md)（Agent 推理架构）。

> RAG 不是"给 LLM 塞一段上下文"这么简单。它是一个信息检索系统和一个生成系统的耦合，两者各自的取舍会在交界处放大或抵消。

## 与向量记忆库的关系

```
RAG系统学习/  ←→  向量记忆库/
  理论笔记           工程代码 + 设计文档
      ↘            ↙
      交叉引用 + 6-工程实战驱动实现
```

- **`RAG系统学习/`** 负责"为什么"和"还有什么"——原理、对比、论文、前沿
- **`向量记忆库/`** 负责"怎么做"——代码实现、设计文档、MCP 集成
- **`6-工程实战/`** 是两者的桥梁：每篇笔记对应一个向量记忆库的实现缺口

## 使用建议

- **理论和工程交替进行**——学完切分策略后，立刻去优化向量记忆库的切分；学完重排序后立刻实现复合评分
- **每个概念笔记末尾写思考题**——强迫自己从"读懂了"走到"能用出来"
- **读完论文后更新关联的概念笔记**——论文是顶点，概念笔记是地基，两者要联动
- **工程实战笔记不是文档**——它们是执行清单，写完一篇意味着对应的代码也写完了
- **MCP 服务器是瓶颈**——所有后续工程任务依赖它，建议第一个动手
