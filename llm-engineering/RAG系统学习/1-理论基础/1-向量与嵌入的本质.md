# 1 · 向量与嵌入的本质

我们需要让计算机理解"这段文字和那段文字意思差不多"。计算机只能做算术。所以不管怎么绕，这个问题的起点只有一个：**你必须把文字变成数字**。

把文字变成数字，最简单的做法是给每个词分配一个编号——"猫"=3721，"狗"=4829。但用这些编号做算术算不出任何有用的东西——3721 和 4829 的平均值不表示"猫狗的平均概念"。这些编号只是标签，不是语义。

把编号改成向量试试。给每个词一个向量，不是一维的数字标签，而是一个列表：`[0.12, -0.34, 0.87, ...]`。现在"相似的词向量应该接近"这个要求有了可操作性——你可以算向量之间的距离。

但问题才刚刚开始。向量从哪来？

一个词的意思是什么？如果你不认识"猫"这个词，给你看一百个包含"猫"的句子——"我养了一只猫"、"猫在沙发上睡觉"、"猫喜欢吃鱼"——你能从这些上下文里猜出"猫"大概是一种宠物。即便你不知道它长什么样，你至少知道它和"狗"出现在相似的句子里（"我养了一只狗"、"狗在沙发上睡觉"），而和"民主"不共享任何上下文。

这就是分布式假设——一个词的意思由它周围的词决定。这个假设不需要任何人工标注，只需要大量文本。它是免费的训练信号。

那怎么把这个假设变成训练目标？最直接的方式：给模型一对句子，告诉它这两个句子是否语义相关。相关的句子，鼓励模型给它们输出相近的向量；不相关的句子，鼓励输出远离的向量。这就是对比学习——整个现代嵌入模型家族（Sentence-BERT、SimCSE、BGE）的训练范式。Word2Vec 的负采样是它的祖辈——只是当时叫"负采样"而非"对比学习"。

> 嵌入模型没有学"每个词什么意思"。它只学了一件事：谁和谁是一伙的，谁和谁不是。所有语义理解都是从这种关系信号中涌现的。

对比学习训练出的向量有一个关键的几何性质：==方向编码语义==。两个向量指向接近的方向，意味着它们在训练中被反复标记为"相似"。"指向"这个行为是纯几何的——你训练模型让正例对的向量夹角变小，负例对的夹角变大。训练完成后，向量的方向就成了语义的代理。

那向量的长度（模长）呢？这取决于训练时的约束。对比学习通常对向量做 L2 归一化（把所有向量压缩到单位球面上），这样余弦相似度就等于点积——计算更简单，检索更快。归一化后，方向是唯一的信息载体。这也是为什么你的向量记忆库和几乎所有主流 RAG 系统都用余弦相似度——它不是被选出来的，它是从对比学习的训练目标中自然产生的。

> 一个 384 维向量不是 384 个独立特征。没有一个维度单独对应"是不是动物"。语义信息分布在整个向量的方向里——这是稠密表示和稀疏表示（如 one-hot）最本质的区别。

从词向量到段落向量，需要把一段话（几百个 token）压缩成一个固定长度的向量。BERT 系的做法是让段落通过 Transformer 的所有层，所有 token 互相注意（自注意力），然后取 `[CLS]` token 的输出或所有 token 的平均。关键是：Transformer 的自注意力让"不"和"好"之间的否定关系、"虽然"和"但是"之间的转折关系都能被捕捉到——最终的向量不只是词向量的机械平均。

回到 RAG。以上所有推导指向同一个结论：你的向量记忆库的检索质量，天花板由嵌入模型决定。如果模型不理解中文、不理解你的领域术语、训练数据的语义空间和你的笔记不重合——后面所有的优化都在错误的坐标上做正确的运算。这也是为什么迁移到 BGE-M3（中文原生支持、领域更匹配）比调任何检索参数都重要——详见 [嵌入模型选型与对比](4-嵌入模型选型与对比.md)。

## 思考题

1. 对比学习的目标函数只要求"正例向量夹角小于负例向量夹角"。这个目标有没有上限？如果你把所有正例对的余弦相似度推到 0.999，会发生什么？
2. 你的笔记里"深模块"和"抽象屏障"在向量空间中应该靠近——但训练数据里这两个词可能从未出现在相近的上下文中。嵌入模型是用什么机制把它们的向量拉近的？如果训练语料里完全没有这两个词，这个机制还工作吗？
