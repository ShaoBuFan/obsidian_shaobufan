# 记忆权重与复合评分

## 问题

纯向量检索只有一个维度：语义相似度。但人的记忆受多种因素影响。一段十年前随口聊的"今天天气不错"，如果碰巧和当前查询语义相关，排名和"你反复强调了三年的核心设计偏好"完全一样。

## 人记住一件事受什么影响

| 维度 | 例子 | 向量检索能表达吗 |
|------|------|:--:|
| **语义相关** | 当前话题和过去某段讨论相关 | ✓ 唯一做的 |
| **情感强度** | 很生气的讨论 vs 平淡的日常 | ✗ |
| **重复次数** | 同一个概念反复聊了十几次 | ✗ |
| **最近性** | 三天前 vs 三年前 | ✗ |
| **刻意标记** | "这个很重要，记住" | ✗ |
| **关联密度** | 这段记忆连着很多其他记忆 | ✗ |

## 方案：复合评分

不是替代语义检索，而是在语义检索后叠加一个 **rerank 阶段**。

```
查询 → 语义检索（top-50）→ 复合评分重排 → 返回 top-5
         向量相似度           + 深度 + 最近 + 重要
```

### 评分公式

```
final_score = semantic_sim × w₁
            + depth_score    × w₂
            + recency        × w₃
            + importance     × w₄
```

权重可调，公式本身不重要，重要的是**每个维度的信号从哪来**。

## 各维度的量化方法

### 1. semantic_sim（语义相似度）

已有的余弦相似度，归一化到 [0, 1]。

### 2. depth_score（深刻程度）

深刻程度本身不是直接可测的数据，需要从其他信号推导：

| 信号 | 含义 | 计算方式 |
|------|------|----------|
| **重复提及** | 同一概念在多次对话中出现 | 检索后统计库里高度相似的 chunk 数量 |
| **对话轮次长度** | 一个话题聊了很久 | 同 session 内连续相关 turns 的数量 |
| **情绪标记** | 当时的情绪强度 | 情感分析模型打分 |
| **显式标记** | 你说"记住这个" | `importance` 字段直接加权 |
| **引用次数** | 后续对话引用的次数 | 类似 PageRank，被引用越多越重 |

最实用的两个：

**重复提及** — 不需要额外数据。检索到一段内容后，用同一向量再搜一次，看返回了多少条相似度 > 0.85 的结果。多 → 说明这个概念你反复讨论过 → 加权。

```python
def depth_score(chunk_vec, collection, threshold=0.85):
    """重复提及越多，深刻度越高"""
    results = collection.query(
        query_embeddings=[chunk_vec],
        n_results=50
    )
    similar_count = sum(
        1 for d in results["distances"][0] if d > threshold
    )
    return min(similar_count / 10, 1.0)  # 归一化，超过10次=满分
```

**显式标记** — 成本最低也最精确。MCP 的 `save_to_memory` 可以指定重要性：

```python
@server.tool("save_to_memory")

    weight_map = {"low": 1, "medium": 3, "high": 5, "critical": 10}
    metadata = {"importance": importance, "importance_weight": weight_map[importance]}
    # ...
```

### 3. recency（最近性）

时间越近，记忆越鲜活。用指数衰减：

```python
import time

def recency_score(timestamp_str, half_life_days=30):
    """艾宾浩斯风格的衰减：每 half_life_days 权重减半"""
    ts = datetime.fromisoformat(timestamp_str).timestamp()
    age_days = (time.time() - ts) / 86400
    return 0.5 ** (age_days / half_life_days)
```

半衰期 30 天意味着：一个月前的对话权重降到 0.5，两个月前 0.25。这是给"最近聊的"一个合理的优先级，但不至于让旧记忆完全消失。

### 4. importance（重要性）

结合多个子信号：

```python
def importance_score(metadata):
    score = 0.0
    # 显式标记
    score += metadata.get("importance_weight", 1) * 0.4
    # 会话长度（长对话 ≈ 深入讨论）
    score += min(metadata.get("turn_length", 0) / 20, 1.0) * 0.3
    # 是不是被保存过（save_to_memory 标记过的）
    score += (1.0 if metadata.get("saved_explicitly") else 0.0) * 0.3
    return min(score, 1.0)
```

## 完整的 rerank 流程

```
查询 → embedding → 向量检索（召回 top-50）
                          │
                          ▼
                  对每条结果计算：
                  ├── semantic_sim   = 余弦相似度（已有）
                  ├── depth_score    = 重复提及数 / 10
                  ├── recency        = 0.5 ^ (age / 30天)
                  ├── importance     = 显式标记 + 会话长度 + 是否保存过
                  │
                  └── final_score = 加权求和
                          │
                          ▼
                  按 final_score 降序，取 top-5 → 返回
```

```python
def compound_search(query, collection, model, top_k=5):
    query_vec = model.encode(query).tolist()

    # 阶段 1：语义召回（多拿一些）
    candidates = collection.query(
        query_embeddings=[query_vec],
        n_results=50
    )

    # 阶段 2：复合评分重排
    scored = []
    for i, (doc, meta, dist) in enumerate(zip(
        candidates["documents"][0],
        candidates["metadatas"][0],
        candidates["distances"][0]
    )):
        semantic = 1 - dist  # ChromaDB 返回的是距离，转相似度
        depth = calc_depth(candidates["embeddings"][0][i], collection)
        recency = calc_recency(meta.get("timestamp", ""))
        importance = calc_importance(meta)

        final = (semantic * 0.6 + depth * 0.2 +
                 recency * 0.1 + importance * 0.1)
        scored.append((final, doc, meta))

    scored.sort(key=lambda x: x[0], reverse=True)
    return scored[:top_k]
```

## 权重调优

没有正确答案，只能靠**你肉眼判断检索结果好不好，然后调**。一个方向性的参考：

| 你对"记忆"的预期 | 建议权重倾向 |
|------------------|-------------|
| 更看重准确匹配 | w₁（语义）↑ |
| 想让 AI 知道你反复思考的东西 | w₂（深度）↑ |
| 想让最近聊的更靠前 | w₃（最近性）↑ |
| 相信自己的显式判断 | w₄（重要性）↑ |

> 这些权重不是一成不变的。比如你可以根据查询类型动态调整：问"帮我回忆一下" → 降低 recency，提高 depth；问"刚才我们说的" → 拉高 recency。

## 进阶：记忆衰减曲线

更精细的做法是**不同类型的内容用不同的衰减速度**：

| 内容类型 | 半衰期 | 逻辑 |
|----------|--------|------|
| 日常对话 | 14 天 | 随口聊的，很快就忘 |
| 技术讨论 | 90 天 | 深入讨论的内容记得久 |
| 显式保存 | 365 天 | 你说"记住"的，应该长期保留 |
| 反复提及的概念 | 不衰减 | 多次出现 = 核心认知 |

## 性能注意

`depth_score` 对每个候选 chunk 再做一次向量查询——如果召回 50 条，就是 50 次额外查询。实际跑起来如果延迟过高：

- **方案 1**：将深度分从实时计算改为写入时预计算。入库时就算好"这个 chunk 在库里有几个相似近邻"，存在 metadata 里，检索时直接读。
- **方案 2**：减少 rerank 候选数（从 50 降到 20），先按语义相似度粗筛，只对这批做昂贵的 depth 计算。
