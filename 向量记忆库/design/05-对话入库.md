# 对话入库

## 数据在哪

Claude Code 的每场对话都存在本地，格式是 JSONL（一行一个事件）：

```
~/.claude/projects/<项目路径hash>/<session-uuid>.jsonl
```

JSONL 中每行是一个事件，`type` 字段标识类型：

| type | 含义 |
|------|------|
| `user` | 用户消息，`message.content` 为文本 |
| `assistant` | Claude 回复，`message.content` 为文本或内容块数组 |
| `file-history-snapshot` | 文件修改备份（用于撤销） |
| `permission-mode` | 权限模式标记 |

关键字段示例：

```json
{
  "type": "user",
  "message": {
    "content": "React Hook 的本质是什么？",
    "role": "user"
  },
  "sessionId": "e4c9e773-...",
  "timestamp": "2026-05-18T00:38:13.340Z"
}
```

## 写入管道

```
JSONL 文件 → 提取对话对 → 切分 → embedding → 存入向量库
```

### 环节 1：提取

从 JSONL 中取出 `user` + `assistant` 连续对，合并为一条"对话记录"。过滤掉纯工具调用消息（assistant 无文本内容的行）。

```python
def extract_turns(jsonl_path):
    turns = []
    current_user = None
    with open(jsonl_path) as f:
        for line in f:
            ev = json.loads(line)
            if ev.get("type") == "user":
                current_user = get_text(ev["message"]["content"])
            elif ev.get("type") == "assistant" and current_user:
                text = get_text(ev["message"]["content"])
                if text:  # 过滤空文本（纯工具调用）
                    turns.append({
                        "user": current_user,
                        "assistant": text,
                        "session": ev.get("sessionId"),
                        "timestamp": ev.get("timestamp")
                    })
                    current_user = None
    return turns
```

### 环节 2：切分 + 元数据标注

每条对话记录带上来源元数据：

```python
{
    "text": "用户问：React Hook 的本质是什么？\nClaude 答：React Hook 的本质是闭包...",
    "metadata": {
        "source": "session/e4c9e773",
        "type": "conversation",
        "date": "2026-05-18",
        "turn_index": 5
    }
}
```

### 环节 3：Embedding + 入库

与笔记写入同一套 pipeline，存入同一个 collection，用 `type` 字段区分来源。入库前做内容哈希去重（见下文）。

## 对话的更新逻辑

对话是**只追加不修改**的，比笔记的更新逻辑更简单——不需要"整文件替换"。

```python
def ingest_new_sessions(processed_ids: set):
    """扫描 JSONL 目录，只处理之前没索引过的会话"""
    jsonl_dir = os.path.expanduser("~/.claude/projects/-home-user-Desktop-WorkSpace/")
    new_sessions = []

    for f in os.listdir(jsonl_dir):
        if not f.endswith(".jsonl"):
            continue
        session_id = f.replace(".jsonl", "")
        if session_id in processed_ids:
            continue
        turns = extract_turns(os.path.join(jsonl_dir, f))
        for turn in turns:
            chunks = chunk_conversation(turn)
            for chunk in chunks:
                if not is_duplicate(chunk["text"]):
                    insert_chunk(chunk)
        new_sessions.append(session_id)
        processed_ids.add(session_id)

    return new_sessions
```

## 三种触发方式

### 方式 A：MCP 显式保存（精确、零噪音）

Claude 在对话中判断哪些内容值得保存，主动调用 `save_to_memory`。工具定义见 [04-MCP服务设计](04-MCP服务设计.md)。

> 优点：只存你确认重要的内容。缺点：依赖 Claude 的判断或你的手动指令。

### 方式 B：Stop Hook 自动保存（全面、有噪音）

每次 Claude 完成回复后，自动将本轮对话写入向量库。

`settings.local.json` 配置：

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "python3 .claude/hooks/ingest_last_turn.py"
          }
        ]
      }
    ]
  }
}
```

Stop hook 从 stdin 收到 `{"session_id": "...", "transcript_path": "...", "cwd": "..."}`，脚本据此增量提取本轮对话并写入。

核心问题：**增量判断**。Stop 每次触发时，JSONL 包含整个会话的全部历史。需要记录"上次处理到第几行"，只处理新增部分。

```python
# .claude/hooks/ingest_last_turn.py
import json, sys, os

hook_input = json.loads(sys.stdin.read())
session_id = hook_input["session_id"]
transcript_path = os.path.expanduser(hook_input["transcript_path"])

# 读取 JSONL，只处理新增行
state_file = f"/tmp/claude_ingest_{session_id}.cursor"
last_line = int(open(state_file).read()) if os.path.exists(state_file) else 0

with open(transcript_path) as f:
    lines = f.readlines()

new_lines = lines[last_line:]
# ... 提取对话对，embedding，写入向量库 ...

# 更新游标
with open(state_file, "w") as f:
    f.write(str(len(lines)))
```

> 优点：全自动，不漏。缺点：噪音大（随口问的"今天天气怎么样"也会入库）。

### 方式 C：混合（推荐）

日常使用 MCP 的 `save_to_memory` 精确保存重要内容。每隔一段时间（每周/每月）批量跑一次全部 JSONL，做一次完整索引。

```bash
python3 向量记忆库/ingest_all_sessions.py \
  --input-dir ~/.claude/projects/-home-user-Desktop-WorkSpace/ \
  --output-db ./vector_memory_db/
```

> 日常精确 + 定期全面 = 最好的平衡。

## 去重

向量库中同一段内容可能被多次写入（保存了又批量导入）。解决：

- **内容哈希去重**：入库前计算 `md5(text)`，检查是否已存在
- **重要性加权**：`save_to_memory` 显式保存的内容标记 `importance: high`，批量导入的标记 `importance: auto`

```python
def insert_chunk(text, metadata):
    content_hash = hashlib.md5(text.encode()).hexdigest()
    existing = collection.get(where={"hash": content_hash})
    if existing["ids"]:
        return  # 跳过重复
    # ... embedding + insert，metadata 中带上 hash
```

## 隐私考虑

JSONL 中可能包含敏感信息。入库前过滤：

- API key 格式（`sk-...`、`ghp_...`）
- `.env` 文件内容
- 只索引 user 和 assistant 的**文本内容**，跳过 `file-history-snapshot` 和工具调用的完整输出

```python
import re
def sanitize(text):
    text = re.sub(r'sk-[a-zA-Z0-9]{20,}', '[REDACTED_API_KEY]', text)
    text = re.sub(r'ghp_[a-zA-Z0-9]{20,}', '[REDACTED_GITHUB_TOKEN]', text)
    return text
```
