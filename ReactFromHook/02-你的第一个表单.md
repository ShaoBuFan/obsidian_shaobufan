# 第二章：你的第一个表单

> [上一章](01-为什么表单这么难.md)我们知道了"为什么"——避免不必要的渲染。这一章我们开始写代码。目标很明确：用最少的 API，写一个能跑、带校验、有 loading 状态的标准表单。

---

## 2.1 打开工具箱：useForm

```tsx
import { useForm } from 'react-hook-form';

const { register, handleSubmit, formState } = useForm();
```

`useForm()` 是一个"工具箱"。调用它，你拿到一堆工具。就像打开工具箱，里面有锤子、螺丝刀、卷尺——你根据需要拿。

本章先认识三个最核心的工具：

| 工具 | 一句话 | 角色类比 |
|------|--------|----------|
| `register` | 告诉表单"这个 input 归我管" | 登记员：给每个字段贴标签、建档案 |
| `handleSubmit` | 收集所有值，校验，交给你 | 收卷人：到时间了，把所有人的答卷收上来 |
| `formState` | 告诉你表单现在的状态 | 体检报告：哪里有问题、正在做什么 |

## 2.2 register——给 input 贴标签

想象你在图书馆登记一批书。每本书你要贴一个标签，告诉系统"这本书叫这个名字，在这个位置"。

```tsx
<input {...register('username')} />
```

`register('username')` 返回一个对象，展开到原生 `<input>` 上：

```tsx
// register 返回的东西大致是：
{
  ref: (node) => { /* 库内部记录这个 DOM 节点，以后从这里读值 */ },
  onChange: (e) => { /* 库内部标记这个字段被修改了 */ },
  onBlur: (e) => { /* 库内部标记这个字段被触碰过了 */ },
  name: 'username',
}
```

**你不需要手动写 ref、onChange、onBlur。** register 帮你包办了。你的工作就是告诉它：这个字段叫什么名字。

> **设计哲学**：为什么 register 设计成展开语法（`{...register()}`）而不是 `<Form.Field name="x">` 这种包装组件？因为展开到原生元素上，让 react-hook-form ==保持透明==——你的 input 还是你的 input，库只是帮你绑了几个属性。你不被锁进一个封闭的组件体系里。换框架？把 `...register()` 删掉就行，input 还是那个 input。

### register 可以带参数

```tsx
register('email', { required: true })  // 第二个参数是配置
```

配置里可以写校验规则，[下一章](03-告诉用户他们填错了.md)详细展开。

## 2.3 handleSubmit——到点了，收卷

```tsx
const onSubmit = (data) => {
  console.log(data);
  // { username: '张三', email: 'zhangsan@mail.com' }
};

<form onSubmit={handleSubmit(onSubmit, onError)}>
```

`handleSubmit` 包装你的业务逻辑，自动做四件事：

1. **阻止页面刷新**——`e.preventDefault()` 你不用写
2. **收集所有 register 过的字段的值**——从 DOM 里读，不是从 state
3. **执行校验**（如果你配置了规则的话）
4. **分支**：校验通过 → 调用 `onSubmit(data)`；校验失败 → 调用 `onError(errors)`（可选），阻止提交

> 为什么要把业务逻辑包装在 handleSubmit 里，而不是自己从 DOM 读值？因为这三个操作——==阻止默认行为、收集数据、执行校验==——在所有表单里完全一样。把它们封装起来，不让你每次都写一遍。这是库该做的事。

## 2.4 formState——表单的体检报告

```tsx
const { formState: { errors, isSubmitting, isDirty } } = useForm();
```

| 属性 | 含义 | 典型用途 |
|------|------|----------|
| `errors` | 校验错误集合 | 显示红色提示"这里填错了" |
| `isSubmitting` | 正在提交中 | 按钮显示 loading，防止重复点击 |
| `isDirty` | 用户修改过任意字段 | 提醒"有未保存的修改" |
| `isValid` | 所有字段校验通过 | 控制提交按钮可点击状态 |

### 关键规则：必须解构

```tsx
// ✅ 正确——解构订阅
const { formState: { errors } } = useForm();

// ❌ 错误——这样拿到的可能是过时的值
const { formState } = useForm();
console.log(formState.errors); // 不保证是最新的
```

**为什么？** react-hook-form 用 JavaScript 的 ==Proxy== 机制来追踪你对哪些状态感兴趣。解构 `formState` 的时候，Proxy 知道"这个组件要 errors，errors 变了就通知它"。不解构直接访问 `formState.errors`，Proxy 不知道你是在 render 里读还是在某个事件回调里读，为了性能不敢每次都重新计算，可能给你旧值。

> 这不是 bug，是刻意的性能优化。大多数组件不需要实时追踪所有 formState 字段。Proxy 实现**按需通知**——只更新你真正关心的部分。在 [后续章节](08-高级模式与技巧.md) 你会看到这个设计思想贯穿整个库。

## 2.5 拼起来：第一个完整的表单

```tsx
import { useForm } from 'react-hook-form';

function SignupForm() {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm();

  const onSubmit = async (data) => {
    // 模拟 API 请求
    await new Promise((r) => setTimeout(r, 1500));
    console.log('提交的数据：', data);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {/* 字段 1 */}
      <label>用户名</label>
      <input {...register('username', { required: '用户名必须填写' })} />
      {errors.username && <span style={{ color: 'red' }}>{errors.username.message}</span>}

      {/* 字段 2 */}
      <label>邮箱</label>
      <input {...register('email', { required: '邮箱必须填写' })} />
      {errors.email && <span style={{ color: 'red' }}>{errors.email.message}</span>}

      {/* 提交按钮 */}
      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? '提交中...' : '注册'}
      </button>
    </form>
  );
}
```

运行这段代码，打开浏览器验证三个行为：
1. 直接点"注册"→ 不提交，红色提示出现
2. 填完两个字段点"注册"→ 按钮显示"提交中..."，1.5 秒后控制台输出数据
3. 输入过程中，页面不卡顿——因为根本没在渲染

---

## 本章要点

- `register` 绑定字段到 DOM，展开到原生 `<input>` 上——你的 input 还是你的 input
- `handleSubmit` 封装了三件事：==阻止刷新、收集数据、校验==、分支回调
- `formState` 提供 `errors`、`isSubmitting` 等状态，**必须解构订阅**（Proxy 按需通知）
- 三个 API 就能写出一个带校验和 loading 状态的生产级表单

## 思考题

1. `formState` 的解构订阅机制用 Proxy 实现。如果一个表单有 50 个字段，10 个组件各订阅了不同的 formState 属性，Proxy 如何知道"谁关心什么"？这个设计有什么潜在的代价？
2. 你在本章的表单中连续快速敲击 10 个字符。react-hook-form 触发了多少次组件渲染？如果是受控组件模式呢？
3. handleSubmit 阻止了浏览器的默认表单提交行为。如果没有 handleSubmit，用原生 `<form>` + `e.preventDefault()` 自己写，你需要额外处理哪些事情？
